# 企微最小 SOP 全流程测试话术

这份文档用于直接在企微自建应用里，从创建产品开始，一路跑到需求关闭和收口查询。

这版按当前场景整理过，不再用 `产品1 / 需求1 / 任务1 / 测试单1 / bug1 / 发布1` 这种占位写法，统一改成你在禅道里能直接搜到的真实主题名。

## 使用方式

建议你直接按顺序一条条发。

默认当前这轮测试统一使用下面这组名称：

- 产品：`测试最小 SOP 流程产品`
- 需求：`测试最小 SOP 流程-新增入口`
- 任务：`测试最小 SOP 流程-研发任务`
- 测试用例：`测试最小 SOP 流程-主流程校验`
- 测试单：`测试最小 SOP 流程-提测单`
- Bug：`测试最小 SOP 流程-登录跳转失败`
- 发布：`测试最小 SOP 流程-发布`

默认还会用到这两个现成编号：

- 执行：`执行4`
- 构建：`构建1`

如果你实际要测的不是 `执行4 / 构建1`，把下面对应位置替换掉就行。

## 这版和之前相比的变化

1. 现在主链路从创建产品直接开始，不再做入口探活。
2. 全部改成当前场景的主题名称，不再写 `产品1 / 需求1 / 任务1 / 测试单1 / bug1 / 发布1`。
3. 产品、需求、任务、测试用例、测试单、Bug、发布，优先都可以直接说名称。
4. 如果名字匹配到多条相似对象，机器人应当回你候选列表，让你回复“第1个需求 / 第1个产品 / 第1个Bug”继续。
5. 提 Bug 和执行用例这两步，当前话术已经按“测试单名 + 用例名”来写，不再要求你手填 `run1`。
