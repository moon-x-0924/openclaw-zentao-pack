# 企微对话式禅道助手架构设计

## 1. 架构目标

- 在不推翻 `openclaw-zentao-pack` 既有能力的前提下，构建一套稳定的企微对话式禅道操作架构。

## 2. 架构分层

- 消息接入层：企微回调、验签、解密、去重
- 会话编排层：意图路由、槽位提取、上下文管理、确认态管理
- 权限与身份层：企微身份识别、禅道账号映射、角色和对象权限校验
- 禅道适配层：统一封装查询、创建、状态更新、关联动作
- 审计与观测层：日志、追踪、错误分类、审计记录

## 3. WeCom 集成模型

- 企业微信消息回调进入接入层
- 接入层解析用户身份和消息内容
- 编排层根据高频路由或通用推理分发给对应 skill 能力

## 4. 禅道适配层

- 复用 `scripts/shared/zentao_client.ts`
- 对外统一暴露查询、创建、更新、关联等能力
- 对错误做统一分类：对象不存在、未登录、权限不足、接口异常

## 5. 身份映射

- 以企微 `userid` 为统一入口身份
- 通过绑定表、账号、姓名等规则匹配禅道账号
- 自动匹配失败时，进入待绑定状态

## 6. 授权检查

- 模型不做最终授权
- 服务层统一校验角色、对象归属和高风险动作资格
- 禅道接口返回结果作为最终执行判定

## 7. 审计设计

- 记录请求来源、企微身份、映射账号、执行动作、目标对象、结果、错误信息
- 高风险操作额外记录确认过程和确认时间

## 8. 会话上下文

- 保存最近对象、最近未完成流程、候选项列表、确认态 token
- 支持 `继续` 和 `取消`

## 9. 幂等、重试与补偿

- 创建类动作需要有幂等防护，避免重复提交
- 查询类失败可直接重试
- 风险动作执行失败要明确区分“未执行”和“结果未知”

## 10. MVP 技术路径

- 先打通回调、身份映射和高频查询
- 再覆盖创建与流转
- 最后补自建应用承接复杂录入和草案确认

## 11. 研发拆分建议

- 消息接入与回调处理
- 会话状态与路由
- 禅道能力封装
- 身份映射与权限校验
- 消息卡片与自建应用页面
- 审计与观测
