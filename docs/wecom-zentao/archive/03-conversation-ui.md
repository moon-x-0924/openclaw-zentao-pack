# 企微对话式禅道助手消息 UI 设计

## 1. 会话界面结构

- 顶部：机器人名称、当前上下文状态
- 中部：消息流
- 底部：输入框、快捷命令入口、最近操作入口

## 2. 消息组件清单

- 结果卡片
- 详情卡片
- 候选项卡片
- 确认卡片
- 失败卡片
- 跳转卡片

## 3. 字段级结构建议

### 结果卡片

- 标题：对象类型 + 核心结果
- 状态：成功 / 失败 / 无结果 / 待确认
- 主体：对象摘要、关键字段、说明
- 操作区：1 到 3 个后续动作

### 确认卡片

- 目标对象
- 当前状态
- 目标状态
- 风险说明
- 确认按钮 / 取消按钮

### 候选项卡片

- 候选对象序号
- 名称 / 编号 / 所属上下文
- 引导文案：回复数字即可选择

## 4. 快捷动作

- 查看详情
- 继续
- 取消
- 去自建应用补齐
- 查看相关对象

## 5. 状态页设计

- 加载态：短提示，避免长时间空白
- 空态：明确告知“无结果”，并给推荐下一步
- 失败态：说明错误原因，给重试或排查建议
- 超时态：提示稍后重试或改查更精确对象
- 权限不足态：说明缺绑定、缺角色或禅道无权限

## 6. 多轮交互规范

- 一次只问一个问题
- 复杂流程尽量分段显示
- 用户使用 `继续` 时，优先恢复最近待办流程
- 用户使用 `取消` 时，清理待确认和待补问状态

## 7. 移动端适配

- 每张卡片控制字段密度
- 候选项优先数字化
- 列表结果只展示最关键字段
- 主动作不超过 3 个

## 8. 桌面端适配

- 可展示更多字段和操作
- 支持更完整的协作摘要
- 适合展示验收、关闭准备等较长结果

## 9. 代表性 UI 场景

- 我的任务列表
- 执行详情
- 发布确认
- 测试准出结论
- 提 Bug 补问
