# 企微对话式禅道助手会话工作流设计

## 1. 意图分类

- 查询类：我的任务、执行、测试单、准出、发布、验收、关闭准备
- 创建类：产品、模块、需求、任务、Bug、测试单、发布
- 流转类：任务、需求、Bug、测试单、发布状态变更
- 管理类：身份绑定、同步、审计查看、帮助

## 2. 槽位体系

- 对象类型：`product`、`project`、`execution`、`story`、`task`、`bug`、`testtask`、`release`
- 对象标识：`id`、名称、标题
- 执行动作：查询、创建、更新、评审、关联、确认
- 上下文字段：当前角色、最近对象、最近未完成流程、确认态 token

## 3. 补问策略

- 一次只补一个关键缺口。
- 优先补 `id`、目标对象、执行范围、最小必要字段。
- 如有多个候选对象，用编号列表返回，支持用户用数字直接选择。

## 4. 多轮对话流程

### 查询类

1. 识别高频命令
2. 尝试用上下文补齐对象
3. 参数足够则直接查询
4. 参数不足则补问
5. 返回结构化结果

### 创建类

1. 识别创建对象
2. 提取已有字段
3. 按最小必要字段补问
4. 参数齐全后调用创建脚本
5. 返回创建结果和后续动作

### 风险流转类

1. 识别目标对象和状态变化
2. 查询对象详情做二次确认
3. 做身份和权限校验
4. 生成确认卡 / 确认文本
5. 用户确认后执行
6. 返回执行回执

## 5. 危险操作确认规则

- 涉及状态流转、关闭、发布、验收结论写入时必须确认。
- 确认信息需要包含：对象、当前状态、目标状态、影响范围、操作者身份。
- 如果用户回复 `继续`，优先承接最近一个待确认流程。
- 如果用户回复 `取消`，终止当前待确认流程。

## 6. 成功反馈规范

- 先给结论，再给对象信息和后续动作。
- 尽量保持默认五段式包装输出。

## 7. 失败反馈规范

- 权限不足：明确说明是身份问题、角色问题还是禅道权限问题。
- 对象不存在：建议改用 ID 或先查列表。
- 参数不足：只追问最关键字段。
- 接口失败 / 超时：给出重试建议和排查方向。

## 8. 权限不足处理

- 未绑定身份时不允许执行敏感动作。
- 身份已绑定但角色不匹配时返回禁止原因。
- 禅道接口返回权限失败时明确回传原因为“禅道未授权”。

## 9. 超时与 API 失败恢复

- 查询失败时允许用户直接重试。
- 执行失败时要区分“未执行成功”和“结果未知”。
- 高风险动作失败后要保留确认态上下文，便于用户继续处理。

## 10. 场景示例

### 示例 A：我的任务

- 用户：`我的任务`
- 系统：识别用户 -> 映射禅道账号 -> 查询我的任务 -> 返回结果

### 示例 B：完成任务

- 用户：`完成任务 33`
- 系统：查询任务详情 -> 校验是否允许完成 -> 返回确认信息
- 用户：`确认`
- 系统：执行状态更新 -> 返回回执

### 示例 C：提 Bug

- 用户：`提个 Bug`
- 系统：缺少产品和标题 -> 逐步补问
- 用户补齐后：创建 Bug -> 返回编号和下一步入口
