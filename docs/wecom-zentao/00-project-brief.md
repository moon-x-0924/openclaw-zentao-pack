# 企微对话式禅道助手项目简述

## 项目名称

- 企微对话式禅道助手

## 业务目标

- 在企业微信中完成禅道高频查询、创建、流转和验收相关操作。
- 复用现有 `openclaw-zentao-pack` 的能力，不从零重复建设。
- 用统一对话体验承接企微机器人与企微自建应用两种入口。

## 目标角色

- 产品
- 研发
- 测试
- 项目经理 / 交付
- 负责人 / 管理层
- 管理员

## 目标平台

- 企业微信机器人
- 企业微信自建应用
- OpenClaw 主会话

## 目标系统

- 禅道
- 企业微信
- OpenClaw 网关 / Skill 运行环境

## 目标交付物

- 产品方案
- 会话工作流设计
- 消息 UI / 卡片设计
- 可落地技术架构
- 研发拆解建议
- 产品 PRD
- HTML 原型与泳道图

## 约束与假设

- 优先复用服务器上已有 `openclaw-zentao-pack` 的技能包、Prompt、路由配置和脚本结构。
- 高风险操作必须具备确认、权限校验和审计能力。
- 复杂字段录入优先通过企微自建应用承接，不强求全部在聊天中完成。

## 已知风险

- 禅道权限与企微身份映射不一致会导致无法直接执行。
- 复杂表单如果全部塞进聊天，会显著降低体验。
- 多对象并存时需要候选项选择，否则容易误操作。

## 待确认问题

- 身份映射采用显式绑定优先还是自动匹配优先。
- 哪些高风险动作必须进入负责人确认。
- 自建应用首期承接的页面边界。
