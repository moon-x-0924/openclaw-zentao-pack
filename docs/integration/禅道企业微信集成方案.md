# 禅道企业微信集成方案

本文档描述当前服务器上 OpenClaw 与禅道 Pack 的企业微信集成实现，重点说明真实生效链路，而不是历史方案草稿。

## 目标

实现以下能力：

- 企业微信消息进入 OpenClaw 后，自动识别是否为禅道请求
- 禅道请求由固定机器人账号登录禅道执行
- 当前企业微信发送人映射为对应禅道用户，作为查询对象或操作者
- 非禅道请求自动回退到上层普通 AI
- 文档、原型、运行说明可通过统一文档入口查看

## 核心设计原则

### 1. 固定机器人账号登录禅道

当前实现不再使用“企微用户本人禅道密码登录”的旧方式，而是统一使用固定机器人账号登录禅道。

真实配置位置：

- `/root/.openclaw/private/zentao.config.json`

说明：

- 禅道连接信息只保存在 `zentao.config.json`
- 不再把禅道账号密码写入 `openclaw.secrets.json` 或 `openclaw.runtime.json` 顶层
- 企微用户身份只用于映射禅道用户，不直接作为登录凭证来源

### 2. 企业微信用户映射为禅道用户

收到企业微信消息后，会优先根据发送人的 `userid` 走数据库映射链路，匹配对应禅道账号。

规则：

1. 先取回调中的企业微信发送人 `userid`
2. 通过数据库映射链路匹配禅道用户
3. 如果匹配成功，则使用映射到的禅道账号作为当前用户上下文
4. 如果匹配失败，则必须提示用户手动提供工号或禅道账号

说明：

- 不再默认尝试用企微用户本人的禅道密码登录
- `user_aliases` 已从配置和代码链路中移除

### 3. 禅道路由优先，普通 AI 兜底

当前企业微信回调不会只处理“我的任务 / 我的 Bug”，而是走统一路由链路：

1. 先对用户消息做文本归一化
2. 优先读取 `agents/modules/intent-routing.yaml`
3. 如果 YAML 未命中，则读取 `agents/openai.yaml` 做 LLM 禅道意图判定
4. 如果判定为禅道请求，则路由到对应脚本执行器
5. 如果判定为非禅道请求，则返回 `should_fallback_to_general_ai: true`

这意味着：

- 高频、明确的业务指令优先由 YAML 命中
- 更自然的口语表达可以由 LLM 兜底判定
- 普通闲聊、天气、问答等不再误走禅道链路

## 当前主要文件

### 回调入口

- `scripts/callbacks/wecom_callback.ts`

职责：

- 读取企业微信消息内容
- 做文本归一化
- 执行 YAML 路由匹配
- 执行 LLM 禅道意图兜底判定
- 调用对应脚本
- 输出结构化结果与 `reply_text`

### LLM 禅道意图判定

- `scripts/callbacks/llm_intent_router.ts`
- `agents/openai.yaml`

职责：

- 当 `intent-routing.yaml` 未命中时，进一步判断用户消息是否属于禅道请求
- 尽量从自然语言中抽取 `execution`、`project`、`testtask`、`bug`、`task` 等参数

### 高频路由来源

- `agents/modules/intent-routing.yaml`

职责：

- 维护高频、稳定、可规则匹配的禅道意图
- 将意图映射到具体脚本和必填参数要求

### 共享回复逻辑

- `scripts/wecom_task_reply.ts`

当前说明：

- 不再只把“我的任务 / 我的 bug”作为独立特判能力维护
- 统一使用通用脚本执行与回复格式输出

## 当前已支持的典型场景

例如：

- 查询我的任务
- 查询我的 Bug
- 查询某个执行 / 迭代是否可以提测
- 查询测试单、发布、验收、关闭准备度
- 创建 Bug
- 其他已补充到 `intent-routing.yaml` 并映射到脚本执行器的能力

自然话术示例：

- `帮我看下我的任务`
- `我的 bug`
- `现在可以提测吗 4`
- `帮我看看 4 号迭代现在是否可以开始测试`

其中前几类一般会被 YAML 直接命中；更自然的说法可由 LLM 兜底识别。

## 当前不再推荐的旧方式

以下内容属于历史方案，不再作为当前主链路：

- 旧目录 `openclaw-zentao-ts` 作为企微主技能入口
- 在 `TOOLS.md` 里手工维护禅道账号密码
- callback 中继续硬编码“我的任务 / 我的 bug”专用分支
- 为了适配自然话术而无限堆叠 YAML 语气词变体
- 默认用发送人本人禅道密码登录

## 结论

当前企业微信集成已经收敛为一条统一链路：

- 企业微信消息进入 OpenClaw
- 识别是否为禅道请求
- 固定机器人账号登录禅道
- 当前企微用户映射为禅道用户上下文
- 禅道请求走脚本执行器
- 非禅道请求回退普通 AI

后续如果继续扩禅道能力，优先补充：

1. `agents/modules/intent-routing.yaml`
2. 对应脚本执行器
3. 相关文档说明

而不是继续新增平行入口或旧式特判分支。
